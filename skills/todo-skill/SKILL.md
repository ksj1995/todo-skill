# Project TODO 管理

跟随项目的待办追踪系统。代码注释 + `PROJECT_TODOS.md` 双重登记。

## 触发场景

**显式触发**：
- `/todo-skill` 或 `/todo`
- "看看待办" / "列出 TODO" / "还有什么没做" / "补全 TODO" / "扫一下 TODO"
- 会话开始时（首次响应前），主动扫一次并报告数量

**隐式触发**（编辑代码时）：
- 检测关键词：`先这样`、`预留`、`待定`、`暂时`、`mock`、`mockdata`、`TODO`、`FIXME`、`后续补充`、`后续处理`、`后续完善`、`后续优化`、`等接口`、`等后端`、`等前端`、`等设计`、`等确认`、`等评审`、`占位`、`占坑`、`待完善`、`待优化`、`待处理`、`待讨论`、`硬编码`、`临时方案`、`临时处理`、`暂缓`、`未完成`、`未实现`、`空实现`、`假数据`、`测试数据`、`开发中`、`建设中`
- 主动询问用户是否登记，并按规范添加标记

## 核心操作

### 1. 扫描代码内 TODO

```bash
# 扫描所有 TODO(预留-xxx) 注释
grep -rn "TODO(预留" src/ --include="*.vue" --include="*.js" --include="*.ts" 2>/dev/null
```

使用 Grep 工具替代 bash grep：
- pattern: `TODO\(预留-(接口|设计|数据|逻辑|依赖)\|[^)]+\):`
- output_mode: `content`
- -n: true

### 2. 读取 `TODO.md`

文件路径：项目根 `TODO.md`

### 3. 比对差异

- **孤儿条目**：md 有，代码无 → 提示清理 md
- **未登记**：代码有，md 无 → 提示补登
- **同步项**：两边都有 → 正常

### 4. 报告格式

```
📋 项目待办（X 项未完成）

🔌 预留-接口（N）
  #20260515-01 [customer] 等 /api/customer/tags
    📍 src/views/customer/Index/index.vue:88
    👤 @ksj  📅 2026-05-15

🎨 预留-设计（N）
  ...

⚠️ 不同步（M）
  孤儿条目：#20260510-03（md 有，代码已删）
  未登记：src/views/xxx.vue:42 (代码有 TODO 注释但 md 无对应)
```

## 添加新 TODO 流程

用户决定添加待办时：

**Step 1: 收集信息**

询问（一次性）：
1. 类型？（接口 / 设计 / 数据 / 逻辑 / 依赖）
2. 模块？（默认从文件路径推断）
3. 说明？
4. 负责人？（默认从 git config user.name 取）

**Step 2: 生成编号**

`#YYYYMMDD-NN`，NN 为当日序号（从 01 起，扫描现有最大值 +1）。

**Step 3: 写代码注释**

格式：`// TODO(预留-类型|模块): 说明 @负责人 YYYY-MM-DD`

注意：
- Vue template 内用 `<!-- TODO(...) -->`
- CSS 内用 `/* TODO(...) */`

**Step 4: 写入 `TODO.md`**

定位对应类型的表格，追加一行：

```markdown
| #20260515-02 | customer | 等 /api/customer/tags 接口 | src/views/customer/Index/index.vue:88 | @ksj | 2026-05-15 | ⏳ |
```

## 完成 TODO 流程

用户说"完成了 TODO #xxx" 或编辑后注释被删时：

1. 确认代码注释已删（grep 验证）
2. 从 `TODO.md` 删除对应行
3. （可选）追加一行到底部"已完成归档"区
4. 提示用户提交：`git add -p` 检查改动

## 关键约束

- **禁止**修改 `src/commons/` 内文件添加 TODO（框架只读）
- **禁止**在 commit message 内加 TODO 标记（commit 应描述已完成的事）
- 跨文件批量操作前**必须**列出影响范围请用户确认
- 编号一旦分配不复用，删除即作废

## 报告"无待办"场景

```
✅ 项目待办：清空
TODO.md 与代码注释均无未完成项）
```

## 与 Claude Code 内置 TaskCreate 区别

| 维度 | 内置 Task | todo |
|------|----------|-------------|
| 持久化 | 当前会话 | git tracked |
| 团队共享 | ❌ | ✅ |
| 代码关联 | ❌ | ✅ 锚点 |
| 适用范围 | 当前任务步骤 | 跨会话/跨人遗留事项 |

**原则**：当前会话内的步骤拆解用 TaskCreate；跨会话的"代码预留"用 todo-skill。
